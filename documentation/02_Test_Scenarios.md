# Test Scenarios
AutomationExercise E-Commerce Website

## Overview
This document outlines the high-level test scenarios identified for the AutomationExercise web application.  
These scenarios represent critical functional flows that must be validated to ensure system stability and business functionality.

The scenarios defined in this document will serve as the foundation for detailed test case development.

---

## Scenario List

### SCN-001 – User Registration

Verify that a new user can successfully create an account using valid information.

Key areas to validate:
- Input field validation
- Required fields
- Account creation confirmation
- Error handling for invalid inputs

---

### SCN-002 – User Login

Verify that an existing user can log into the application using valid credentials.

Key areas to validate:
- Valid login authentication
- Invalid credential handling
- Session creation
- Login error messages

---

### SCN-003 – Product Browsing

Verify that users can browse available products and access product details.

Key areas to validate:
- Product listing visibility
- Product detail page accessibility
- Navigation functionality

---

### SCN-004 – Add Product to Cart

Verify that users can add products to the shopping cart.

Key areas to validate:
- Add-to-cart functionality
- Cart item count updates
- Product information accuracy in cart

---

### SCN-005 – Cart Management

Verify that users can manage items inside the cart.

Key areas to validate:
- Item removal
- Cart updates
- Quantity adjustments
- Cart total calculation

---

### SCN-006 – Checkout Process

Verify that users can proceed through the checkout flow.

Key areas to validate:
- Shipping information entry
- Order summary accuracy
- Checkout navigation

---

### SCN-007 – Payment Submission

Verify that users can submit payment information and complete an order.

Key areas to validate:
- Payment field validation
- Order confirmation
- Payment submission flow

---

### SCN-008 – Contact Form Submission

Verify that users can submit inquiries through the contact form.

Key areas to validate:
- Email validation
- Required fields
- Message submission confirmation

---

### SCN-009 – Account Deletion

Verify that users can delete their account.

Key areas to validate:
- Account deletion confirmation
- Data removal
- Post-deletion access restrictions
